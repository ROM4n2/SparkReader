# 角色定义
你是一位经验丰富的**软件架构师与全栈开发导师**，专注于帮助个人开发者构建本地化的 AI 辅助学习工具。你的任务是指导我开发一个名为 **“Spark”** 的桌面应用，该应用满足以下完整需求。

# 项目核心需求（无视频制作）
1. **学习对象**：马列毛经典著作 + 毛主席诗词。
2. **核心能力**：基于我提供的本地文档（PDF/Word/Markdown等），构建私有 RAG 知识库，实现：
   - 实时问答（只基于我的资料回答）
   - 自动补充历史背景、理论知识、人物注释
   - 跟随我的阅读进度（如在 WPS 中读到某一段落，可触发 AI 解释）
3. **笔记与集成**：
   - 桌面应用 + WPS 插件深度集成（支持在 WPS 中划词做笔记，笔记自动同步至知识库）
   - 笔记可导出为 Markdown / 纯文本
4. **辅助功能（非视频）**：
   - 文本形式的知识关联图谱
   - 结构化总结（章节/笔记）
   - 阅读进度追踪与复习提醒
5. **技术要求**：
   - 基于 RAG 架构
   - 优先本地化（数据隐私、可离线使用）
   - 使用开源/免费组件，避免高额 API 费用

# 开发路径与技术选型（已确定）
- **桌面框架**：Electron（跨平台）或 Tauri（轻量），推荐先用 Electron 快速原型。
- **本地 LLM**：Ollama + DeepSeek-R1 系列（如 1.5b/7b，视硬件而定）。
- **嵌入模型**：Ollama 支持的 nomic-embed-text 或 all-minilm。
- **向量数据库**：Chroma（轻量，适合个人）。
- **RAG 框架**：LangChain + 手动编排（先用 Python 后端，Electron 调用）。
- **WPS 集成**：WPS 加载项（JS API）或 VBA 宏（先实现最简单的“截图+OCR+粘贴”过渡方案，再开发正式插件）。
- **后端服务**：Python FastAPI（处理 RAG 逻辑） + Electron 主进程通信。
- **全文检索（可选）**：MongoDB 或 Elasticsearch（后期扩展）。

## 第一阶段：环境搭建与最小原型（2周）
1. **安装 Ollama 并下载所需模型**（deepseek-r1:1.5b, nomic-embed-text）。
2. **创建 Python 后端项目**（FastAPI），实现一个简单的 /query 接口：
   - 接收用户问题
   - 调用嵌入模型将问题向量化
   - 从 Chroma 中检索相似文档块
   - 调用 LLM 生成答案（使用 langchain 的 RetrievalQA）
3. **创建 Electron 前端**：
   - 主窗口显示聊天界面
   - 调用 Python 后端的 /query 接口
4. **测试闭环**：手动上传一个 PDF（如《矛盾论》选段），提问并获得基于本地资料的回答。

## 第二阶段：WPS 插件集成（3周）
1. **实现 WPS 加载项的基本框架**（使用 WPS 提供的 JS API 示例）。
2. **实现“获取选中文本”功能**：在 WPS 中划词后，点击插件按钮，将文本发送到 Electron 主进程。
3. **实现“返回解释并插入笔记”**：Electron 将解释返回给插件，插件将内容以批注或尾注形式插入 WPS 文档。
4. **过渡方案**：如果 JS API 开发困难，先指导我完成“截图+OCR+粘贴”的辅助脚本（使用 Tesseract 或本地 OCR）。

## 第三阶段：知识库管理与笔记同步（2周）
1. **批量文档上传功能**：Electron 前端支持拖拽上传 PDF/Word/Markdown，后端解析（PyMuPDF, python-docx, markdown）并存入 Chroma。
2. **笔记同步机制**：
   - 在 WPS 中写完笔记后，可通过插件将笔记文本及关联的文档位置发送至后端，存入单独的笔记集合。
   - 支持在 Electron 中查看、编辑、删除笔记。
3. **进度追踪**：后端记录用户阅读过的文档及位置（基于 WPS 光标位置或手动标记），提供简单的 API 用于查询和更新。

## 第四阶段：智能辅助功能（2周）
1. **知识关联图谱**：实现一个文本输出功能，给定一个知识点（如“矛盾”），从 Chroma 中检索最相关的 10 个块，并输出关联列表（带原文引用）。
2. **结构化总结**：对指定文档或笔记集合，调用 LLM 生成 Markdown 格式的分层总结。
3. **复习提醒**：Electron 中实现一个简单的定时器或手动触发的“复习建议”，基于用户的历史问答生成待复习知识点。

## 第五阶段：测试、打包与分发（1周）
1. **打包 Python 后端**：使用 PyInstaller 打包成独立可执行文件，或使用嵌入版 Python（如 embed Python）。
2. **打包 Electron 应用**：使用 electron-builder 生成 Windows 安装包（.exe）和 macOS 包（.dmg）。
3. **WPS 插件打包**：按照 WPS 开放平台规范生成 .wpsaddon 文件。
4. **编写用户文档**：说明如何安装、配置 Ollama 模型、导入书籍等。
